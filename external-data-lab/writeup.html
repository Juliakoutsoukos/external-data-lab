AI Tool Reflection

For this lab, I used Copilot with two different models and compared how well they handled (1) load order for Tailwind + Alpine + module scripts and (2) fetch() code using two API endpoints. One model was better at explaining *why* Alpine must load before my app code (so window.pokedexApp exists when the page initializes), and it suggested splitting the API logic into a separate file to keep code readable. The other model produced faster “copy/paste” code, but it initially mixed defer scripts with modules incorrectly and didn’t handle missing sprite data until I prompted it with the error I saw. The most helpful prompts were: “Show me a clean file structure with multiple JS files and correct script order” and “Render two different endpoints on the same page with Tailwind cards.” Overall, AI sped up scaffolding, but I still had to debug and adjust details like endpoint fields, null images, and module import paths.
